# Selección tecnológica con Angular

Elegir Angular implica analizar requisitos técnicos, organización y ciclo de vida del producto.

## Cuándo elegir Angular

- Equipos medianos o grandes que necesitan **convenciones fuertes** y tooling integrado.
- Proyectos con **requisitos de accesibilidad, internacionalización y seguridad** desde el inicio.
- Plataformas que requieren **soporte de largo plazo** y una hoja de ruta pública.

## Comparativa rápida

| Aspecto | Angular | React | Vue |
| --- | --- | --- | --- |
| Arquitectura | Opinada, modular | Flexible, librería | Progresiva |
| Tooling oficial | CLI, Material, DevTools | Create React App (deprecado), Vite | CLI, Vite |
| Escalabilidad | Alta | Depende de elecciones | Media |
| Documentación | Centralizada | Distribuida | Centralizada |

## Capacidad organizacional

- **Governance**: define un *Design System* y directrices de arquitectura.
- **Dev Experience**: proporciona plantillas y generadores personalizados (`schematics`).
- **Formación**: planifica workshops sobre signals, builder y testing moderno.

## Indicadores de éxito

- Tiempo medio de onboarding < 2 semanas.
- Cobertura de pruebas > 75 % con pipelines automatizados.
- Métricas de Core Web Vitals en verde (LCP, FID, CLS).
